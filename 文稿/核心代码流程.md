## 5. 贪心主算法

```cpp
auto runGreedy = [&](std::mt19937& rng) -> std::vector<int> {
    // ① 重置状态（每次重启）
    std::fill(usedCand.begin(), usedCand.end(), false);
    std::fill(covState.begin(), covState.end(), false);
    std::fill(jCnt.begin(), jCnt.end(), 0);
    std::fill(jSat.begin(), jSat.end(), false);
    int remaining = numJ;    // 还有多少 j-子集未满足
    std::vector<int> res;

    std::vector<bool> coversThis(numSubs, false);  // ← 每轮每候选都要 fill 一次！
    std::vector<int>  bestCands;

    while (remaining > 0) {  // ② 外层贪心循环（每轮选一个 k-组）
        int bestScore = -1;
        bestCands.clear();

        // ③ 内层：对每个未用候选评分（全量 O(numJ×C(j,s))）
        for (int c = 0; c < (int)kCandidates.size(); ++c) {
            if (usedCand[c]) continue;

            // 先用 bool 数组标记此候选覆盖哪些 s-子集
            std::fill(coversThis.begin(), coversThis.end(), false);  // ← O(numSubs) 重置
            for (int si : kCoversS[c]) coversThis[si] = true;

            // 再遍历所有 j-子集评分
            int score = 0;
            for (int i = 0; i < numJ; ++i) {          // ← C(n,j) 次！
                if (jSat[i]) continue;
                int need = minCover_ - jCnt[i];
                int newCount = 0;
                for (int si : jToSubs[i]) {            // ← 每个 j-子集再遍历 C(j,s) 次
                    if (coversThis[si] && !covState[si]) ++newCount;
                }
                score += std::min(newCount, need);
            }
            // 收集最高分候选
            if (score > bestScore) { bestScore=score; bestCands.clear(); bestCands.push_back(c); }
            else if (score == bestScore) { bestCands.push_back(c); }
        }

        if (bestCands.empty() || bestScore == 0) break;

        // ④ 从同分候选中随机选一个
        std::uniform_int_distribution<int> dist(0, (int)bestCands.size()-1);
        int chosen = bestCands[dist(rng)];

        usedCand[chosen] = true;
        res.push_back(chosen);

        // ⑤ 选中后更新：仍然是全量扫描！
        for (int si : kCoversS[chosen]) covState[si] = true;
        for (int i = 0; i < numJ; ++i) {              // ← 再一次全量 O(numJ) 扫描！
            if (jSat[i]) continue;
            int cnt = 0;
            for (int si : jToSubs[i]) if (covState[si]) ++cnt;  // 重新累计
            jCnt[i] = cnt;
            if (cnt >= minCover_) { jSat[i] = true; --remaining; }
        }
    }
    return res;
};

// ⑥ 多次重启（500次）
const int NUM_RESTARTS = (kCandidates.size() <= 200) ? 500 : 1;
std::mt19937 rng(42u);
std::vector<int> resultIdx;
for (int r = 0; r < NUM_RESTARTS; ++r) {
    auto candidate = runGreedy(rng);        // 每次传同一个 rng（连续推进）
    if (resultIdx.empty() || candidate.size() < resultIdx.size())
        resultIdx = std::move(candidate);
}
```

### 贪心算法解释

#### 算法实现的核心目标
这段代码的目标是：

> 从很多个 `k`-候选组里，**每次挑一个当前“最有用”的候选组**，直到所有 `j`-子集都满足覆盖要求。

这里“最有用”的意思不是随便定义的，而是：

> **谁能让当前还没满足的 `j`-子集，获得最多“有效新增覆盖”，就优先选谁。**


### 1.算法的核心状态：

1. `usedCand`
表示：

> 哪些 `k`-候选组已经被选过了

- `false`：还没选
- `true`：已经选进结果了

作用是防止重复选同一个候选组。

2. `covState`
表示：

> 当前全局每个 `s`-子集是否已经被覆盖

- `covState[si] = true`：说明编号为 `si` 的 `s`-子集已经被某个已选中的 `k`-组覆盖了
- `false`：还没覆盖

它反映的是**全局覆盖状态**。

3. `jCnt`
表示：

> 每个 `j`-子集当前已经有多少个相关的 `s`-子集被覆盖了

也就是：

jCnt[i] = 第 i 个 j子集 当前已覆盖数

4. `jSat`
表示：

> 每个 `j`-子集是否已经满足要求

满足条件是：

jCnt[i] >= minCover_

如果满足了，就不用再重点管它了

5. `remaining`
表示：

> 还有多少个 `j`-子集没有满足

6. `res`
表示：

> 这次贪心运行最终选中了哪些候选组

7. `coversThis`
表示：
> 当前正在评分的某个候选组，它覆盖了哪些 `s`-子集

8. `bestCands`
表示：
> 当前这一轮评分最高的所有候选组编号

### 2.算法流程

0. ：初始化

重置所有状态，表示“从零开始做一次贪心构造”。

1. ：进入 while 循环

只要还有 `j`-子集没满足，就继续。

2. ：枚举所有还没使用的候选组

给每个候选组打分。

3. ：找到分数最高的候选组

如果有多个并列第一，就先都记下来。

4. ：随机选一个最高分候选

把它加入当前解里。

5. ：更新全局覆盖状态

包括：

- 新覆盖了哪些 `s`-子集
- 各个 `j`-子集现在覆盖了多少
- 哪些 `j`-子集已经满足

6. ：重复

直到所有 `j`-子集满足，或者再也找不到有用的候选。


## 3.具体实现

### **1. 每次重启先清空状态**

```cpp
std::fill(usedCand.begin(), usedCand.end(), false);
std::fill(covState.begin(), covState.end(), false);
std::fill(jCnt.begin(), jCnt.end(), 0);
std::fill(jSat.begin(), jSat.end(), false);
int remaining = numJ;
std::vector<int> res;
```

因为后面有多次随机重启，所以每次都必须恢复到初始状态。

### **2.外层 while 循环**

```cpp
while (remaining > 0)
```

>只要还有 `j`-子集没满足，就必须继续选新的 `k`-候选组

### **3.这一轮先准备找“最佳候选**

```cpp
int bestScore = -1;
bestCands.clear();
```
- `bestScore`：记录目前看到的最高分
- `bestCands`：记录所有达到最高分的候选编号

### **4.内层循环**

#### **5. 内层循环 -- 如何给每个候选打分** 
```cpp
for (int c = 0; c < (int)kCandidates.size(); ++c) {
    if (usedCand[c]) continue;
```

枚举所有还没选过的 `k`-候选组，计算它现在有多大价值。

#### **一. 先标出这个候选覆盖哪些 `s`-子集**

```cpp
std::fill(coversThis.begin(), coversThis.end(), false);
for (int si : kCoversS[c]) coversThis[si] = true;
```
假设当前看的候选是 `c`，它覆盖的 `s`-子集编号是：

kCoversS[c] = {2, 5, 8}

那执行后：

- `coversThis[2] = true`
- `coversThis[5] = true`
- `coversThis[8] = true`

其余都还是 `false`

这样后面判断某个 `s`-子集是否被这个候选覆盖时，就可以直接 O(1) 查：coversThis[si]


#### **二.开始算这个候选的分数**

```cpp
int score = 0;
for (int i = 0; i < numJ; ++i) {
    if (jSat[i]) continue;
```

> 这个候选组对所有“还没满足的 `j`-子集”分别能贡献多少，把这些贡献加总起来，就是它的总分。
已经满足的 `j`-子集直接跳过


#### **三.看某个 `j`-子集还差多少**

```cpp
int need = minCover_ - jCnt[i];
```

意思是：
> 第 `i` 个 `j`-子集距离满足要求，还差多少个覆盖
例如：
- `minCover_ = 3`
- 当前 `jCnt[i] = 1`
那么：说明它还差 2 个有效覆盖。


#### **四.计算当前候选对这个 `j`-子集能新增多少覆盖**

```c++
int newCount = 0;
for (int si : jToSubs[i]) {
    if (coversThis[si] && !covState[si]) ++newCount;
}
```

`jToSubs[i]` 表示：

> 第 `i` 个 `j`-子集包含哪些 `s`-子集

现在我们遍历这些 `s`-子集，看其中有多少满足：

- `coversThis[si] == true`：当前候选能覆盖它
- `covState[si] == false`：它之前还没有被覆盖

如果同时满足，说明：

> 当前候选对这个 `j`-子集带来了一个“新的有效覆盖”

于是 `newCount++`


#### **五.这个 `j`-子集对总分贡献多少

```cpp
score += std::min(newCount, need);
```

为什么不是直接加 `newCount`？
因为：
- 一个 `j`-子集只要达到 `minCover_` 就够了
- 超过部分不应该继续算很高价值
那你真正需要的只有 1 个，多出来那 3 个对“满足约束”没有额外价值。

#### **跳出内层循环**


### **6.最佳候选

当一个候选的分数算完后，就和当前最好分数比较。
```cpp
if (score > bestScore) {
    bestScore = score;
    bestCands.clear();
    bestCands.push_back(c);
}
else if (score == bestScore) {
    bestCands.push_back(c);
}
```

### 7.什么时候停止

```cpp
if (bestCands.empty() || bestScore == 0) break;
```

### 8.同分候选里随机选一个

```cpp
std::uniform_int_distribution<int> dist(0, (int)bestCands.size()-1);
int chosen = bestCands[dist(rng)];
```

原因：增加搜索多样性，减少陷入局部最优的概率

### 9.选中一个候选后，怎么更新状态

选中后先做两件事：

```cpp
usedCand[chosen] = true;  
res.push_back(chosen);
```

- 这个候选以后不能再选了
- 把它加入本次贪心构造出来的解里

### 10.更新全局已覆盖的 `s`-子集

```cpp
for (int si : kCoversS[chosen]) covState[si] = true;
```

### 11. 重新计算每个 `j`-子集当前覆盖数

```cpp
for (int i = 0; i < numJ; ++i) {
    if (jSat[i]) continue;
    int cnt = 0;
    for (int si : jToSubs[i]) if (covState[si]) ++cnt;
    jCnt[i] = cnt;
    if (cnt >= minCover_) { jSat[i] = true; --remaining; }
}
```

> 因为我们刚刚选了一个新的候选组，所以现在要重新检查每个还没满足的 `j`-子集，到底已经覆盖了多少。

对于第 `i` 个 `j`-子集：

- 遍历它对应的所有 `s`-子集
- 看其中多少已经在 `covState` 中被标记为 true
- 这就是它现在的覆盖数 `cnt`

然后：

- 更新 `jCnt[i]`
- 如果 `cnt >= minCover_`，说明这个 `j`-子集终于满足了
- 则 `jSat[i] = true`
- 同时 `remaining--`

### 算法问题

```
评分阶段：C(n,k)候选 × [C(k,s)设标记 + C(n,j)×C(j,s)扫j-子集]
        = 480K × [7 + 480K × 7] = 480K × 3.36M = 1.6万亿 次操作

更新阶段：C(n,j) × C(j,s) = 480K × 7 = 3.36M 次

每轮总计：~1.6万亿，运行时间 > 30分钟
```

## 构建倒排索引 sCoversList

```cpp
std::vector<std::vector<int>> sCoversList(numSubs);  // 长度 = s-子集总数
for (int i = 0; i < numJ; ++i)
    for (int si : jToSubs[i])
        sCoversList[si].push_back(i);
```

具体执行（n=7, j=s=5）：

```
外层循环 i 从 0 到 20（遍历所有 j-子集）：

i=0：jToSubs[0]={0} → sCoversList[0].push_back(0)

i=1：jToSubs[1]={1} → sCoversList[1].push_back(1)

...

i=20：jToSubs[20]={20} → sCoversList[20].push_back(20)
```

结果：`sCoversList[si] = {si}`（因为 j=s，每个 s-子集只属于一个 j-子集）。

对 j=6, s=5（示例5）时的效果更明显：

```
j-子集{1,2,3,4,5,6} 包含的 s-子集编号有 C(6,5)=6 个
→ sCoversList[每个这6个s-子集编号].push_back(j-子集{1,2,3,4,5,6}的编号)
```

### 新代码（正向枚举）：

```cpp
std::vector<std::vector<int>> kCoversS(kCandidates.size());
for (int c = 0; c < (int)kCandidates.size(); ++c)
    for (auto& sub : enumerate(kCandidates[c], s_)) {  // ← 只枚举 k-候选自身的 C(k,s) 个 s-子集
```

```
c=0，kg={1,2,3,4,5,6}：

    {1,2,4,5,6} → 查 subIndex → 编号6 → 记录6

kCoversS[0] = {0,1,3,6,10,15}   ← 6个，花6次操作
kCoversS[1] = {0,1,2,6,10,15}   ← {1,2,3,4,5,7}缺6，花6次操作

```

## 贪心评分函数 `computeScoreImpl`（新代码新增）

旧代码：没有独立的评分函数，评分逻辑内嵌在 `runGreedy` 循环内。

```cpp
auto computeScoreImpl = [&](const std::vector<int>& kSubs,   // k-候选的 s-子集编号列表
                             const std::vector<bool>& cov,    // 当前哪些 s-子集已覆盖
                             const std::vector<int>&  jC,     // 当前每个 j-子集的覆盖计数
                             const std::vector<bool>& jS)     // 当前哪些 j-子集已满足
    -> int {
    dirtyList.clear();                    // 清空脏标记列表（不重置整个 jNewCount 数组）

    // 第一个小循环：找出"加入此候选后，哪些 j-子集会新增覆盖"
    for (int si : kSubs) {               // 只遍历该候选的 C(k,s) 个 s-子集（最多35个）
        if (cov[si]) continue;           // 跳过已覆盖的（加入也无新贡献）
        for (int jIdx : sCoversList[si]) { // 通过倒排索引，找到包含 si 的 j-子集
            if (jS[jIdx]) continue;      // 跳过已满足的（已经不需要新覆盖）
            if (jNewCount[jIdx] == 0)    // 第一次影响到该 j-子集？
                dirtyList.push_back(jIdx); // 记录到脏列表
            ++jNewCount[jIdx];           // 计数：此候选能给该 j-子集增加几个 s-子集覆盖
        }
    }

    // 第二个小循环：对所有受影响的 j-子集汇总分数，然后归零
    int score = 0;
    for (int jIdx : dirtyList) {
        // 贡献 = min(新增的覆盖数, 该 j-子集还差多少才满足)
        score += std::min(jNewCount[jIdx], minCover_ - jC[jIdx]);
        jNewCount[jIdx] = 0;             // 归零（用"脏标记"替代全量 memset）
    }
    return score;
}
```

关键设计—脏标记（dirty list）：

`jNewCount` 是一个长度为 numJ 的数组，全局初始为 0。每次调用只修改了 `dirtyList` 里的那些位置，用后归零，不需要 `std::fill(jNewCount, 0)` 来重置整个数组（那会是 O(numJ) 操作）。

具体执行（k-候选={1,2,3,4,5,6}，kSubs={0,1,3,6,10,15}，j=s=5，minCover=1）：
si=0（{1,2,3,4,5}），未覆盖：
```
si=0（{1,2,3,4,5}），未覆盖：
  sCoversList[0]={0}（j-子集{1,2,3,4,5}包含它）
  jNewCount[0]=1，dirtyList=[0]

si=1（{1,2,3,4,6}），未覆盖：
  sCoversList[1]={1}
  jNewCount[1]=1，dirtyList=[0,1]

...（共6个 si，各自影响1个 j-子集）

汇总：
  dirtyList=[0,1,3,6,10,15]
  score = Σ min(1, 1-0) = 6
  归零 jNewCount[0]=jNewCount[1]=...=0
```

## 核心算法

```cpp
auto runLazyGreedy = [&](unsigned seed) -> std::vector<int> {
    std::mt19937 rng_local(seed);                       // 每次重启独立的随机引擎
    std::uniform_int_distribution<int> tbDist(0, INT_MAX); // 随机同分打破器

    // ① 重置状态
    std::fill(usedCand.begin(), usedCand.end(), false);
    std::fill(covState.begin(), covState.end(), false);
    std::fill(jCnt.begin(),    jCnt.end(),    0);
    std::fill(jSat.begin(),    jSat.end(),    false);
    int remaining = numJ;
    std::vector<int> res;

    // ② 初始全量评分，建优先队列
    std::priority_queue<T3> pq;  // max-heap，T3 = tuple<score, tieBreak, idx>
    for (int c = 0; c < (int)kCandidates.size(); ++c)
        pq.push({scoreForCand(c),    // 用新评分函数，O(C(k,s)×avg_sCoversList)
                 tbDist(rng_local),  // 随机打破器，使同分候选顺序在每次重启随机化
                 c});

    // ③ 懒惰贪心主循环
    while (remaining > 0 && !pq.empty()) {
        auto [topScore, tb, c] = pq.top(); pq.pop();  // 取堆顶（缓存的最高分）

        if (usedCand[c]) continue;   // 已经选过，跳过

        // ④ 惰性验证：重新计算真实分数（O(C(k,s)×avg_sCoversList)）
        int real = scoreForCand(c);
        if (real < topScore) {
            // 分数已降低（有其他候选被选过，覆盖了部分 s-子集）
            // → 带新分数重新入堆，不立刻使用
            pq.push({real, tbDist(rng_local), c});
            continue;
        }
        if (real == 0) break;        // 没有任何候选能带来新覆盖，退出

        // ⑤ 验证通过，选中此候选
        usedCand[c] = true;
        res.push_back(c);

        // ⑥ Layer 2 增量更新（最关键优化：O(C(k,s)×avg_sCoversList) 替代 O(numJ)）
        for (int si : kCoversS[c]) {             // 只遍历 C(k,s) 个 s-子集（最多35个）
            if (covState[si]) continue;
            covState[si] = true;                 // 标记此 s-子集已覆盖

            for (int jIdx : sCoversList[si]) {   // 通过倒排索引，找到受益的 j-子集
                if (jSat[jIdx]) continue;
                if (++jCnt[jIdx] >= minCover_) { // 累计：此 j-子集的 s-子集覆盖数达到要求？
                    jSat[jIdx] = true;
                    --remaining;
                }
            }
        }
        // 注意：不需要重置任何大数组！下一次 scoreForCand 会通过 covState 感知最新状态
    }
    return res;
};

// ⑦ 多次重启，每次用不同 seed（确保每次重启路径真正不同）
int numRestarts = ((int)kCandidates.size() <= 200)  ? 500 :
                  ((int)kCandidates.size() <= 2000) ? 50  : 10;
std::vector<int> bestIdx;
for (int r = 0; r < numRestarts; ++r) {
    auto cand = runLazyGreedy(42u + (unsigned)r);   // 每次不同种子
    if (bestIdx.empty() || cand.size() < bestIdx.size())
        bestIdx = std::move(cand);
}
```

具体执行（示例1，n=7, k=6, j=s=5, numCands=7）：

```
初始评分（7个候选全部评分）：
  候选{1,2,3,4,5,6}：score=6，heap入 (6, rand1, 0)
  候选{1,2,3,4,5,7}：score=6，heap入 (6, rand2, 1)
  ...（7个都是6分，因为每个恰好覆盖6个j-子集）

第1轮循环：
  堆顶 = (6, 最大rand, 候选c)，取出
  惰性验证：real = scoreForCand(c) = 6（与topScore相同）→ 验证通过
  选中候选c，res=[c]
  增量更新：kCoversS[c]={...6个编号...}
    si=0（{1,2,3,4,5}）→ covState[0]=true → jCnt[0]++ = 1 ≥ 1 → jSat[0]=true, remaining=20
    si=1 → ... remaining=19
    ...→ remaining=15（选中的候选覆盖了6个j-子集）

第2轮循环：
  堆顶 = (6, rand, 另一个候选c')
  惰性验证：real = scoreForCand(c')
    → kCoversS[c']的某些si已在covState里 → 这些si被skip
    → real 可能 < 6（因为有些被覆盖了）
  若 real < topScore=6 → 带新分 (real, rand, c') 重入堆
  下一次从堆顶取 → 再验证...
  直到找到 real == topScore 的候选
```
新代码每轮成本：
```
初始评分：7 × O(C(k,s)×avg) = 7 × 6 ≈ 42 次
每次惰性验证：O(C(k,s)×avg) = 6 次
增量更新：O(C(k,s)×avg_sCoversList) = 6 × 1 ≈ 6 次
总计：< 100 次操作（vs 旧代码每轮数千次）
```

## 第七步：模拟退火（大规模时的新路径）

### `saAdd` / `saRemove` lambda（核心是引用计数）：
```cpp
auto saAdd = [&](const std::vector<int>& kSubs) -> int {
    int gain = 0;
    for (int si : kSubs) {          // 遍历该 k-组的 C(k,s) 个 s-子集编号
        if (++covRef[si] == 1) {    // covRef[si]++ 后等于1 → 从0变1 → 新被覆盖
            for (int jIdx : sCoversList[si]) {      // 倒排索引找受益 j-子集
                if (!jSatSA[jIdx] && ++jCntSA[jIdx] == minCover_) {
                    jSatSA[jIdx] = true;  // 该 j-子集恰好达到满足阈值
                    ++gain;
                }
            }
        }
        // 如果 covRef[si] 从1→2，说明多个组覆盖同一个 si，不需要额外操作
    }
    satisfiedSA += gain;
    return gain;
};
```

`covRef[si]` 是引用计数：记录有多少个当前解中的 k-组覆盖了 s-子集 si。只有从 0→1（第一次覆盖）才触发 j-子集更新。

```cpp
auto saRemove = [&](const std::vector<int>& kSubs) -> int {
    int loss = 0;
    for (int si : kSubs) {
        if (--covRef[si] == 0) {    // covRef[si]-- 后等于0 → 从1变0 → 失去覆盖
            for (int jIdx : sCoversList[si]) {
                if (jCntSA[jIdx] > 0) {
                    --jCntSA[jIdx];
                    if (jSatSA[jIdx] && jCntSA[jIdx] < minCover_) {
                        jSatSA[jIdx] = false;  // 满足状态丢失
                        ++loss;
                    }
                }
            }
        }
        // 如果 covRef[si] 从2→1，其他组仍覆盖，不触发 j-子集更新
    }
    satisfiedSA -= loss;
    return loss;
};
```

### SA 主循环三种动作详解：

```cpp
for (int step = 0; step < SA_STEPS; ++step, T *= ALPHA) {
    // T *= ALPHA：每步温度乘以 ALPHA（<1），温度缓慢降低

    int action = actD(rng);   // 随机选 0(Add)/1(Remove)/2(Swap)

    if (action == 0) {  // === Add ===
        auto ng = randKGroup();           // 随机生成新 k-组（从 samples_ 随机选 k 个）
        auto ns = getKSubIndices(ng);     // 计算其 s-子集编号
        const int satB = satisfiedSA;
        saAdd(ns);                        // 试探性加入，更新 covRef/jCntSA/jSatSA
        const double delta = (satisfiedSA - satB) * 100.0 - 1.0;
        // delta = sat增益×100 - 1（组数增加的代价）
        // 若 sat增加了：delta>0，一定接受
        // 若 sat不变：delta=-1<0，以 exp(-1/T) 概率接受（温度高时容易接受）
        if (delta < 0 && uProb(rng) > std::exp(delta / T)) {
            saRemove(ns);   // 拒绝→回滚（撤销 saAdd 的影响）
        } else {
            finalGroups.push_back(ng);  // 接受→正式加入
            groupKSubs.push_back(ns);
        }

    } else if (action == 1 && szNow > 1) {  // === Remove ===
        int gi = gD(rng);                   // 随机选一个现有组
        const int satB = satisfiedSA;
        saRemove(groupKSubs[gi]);           // 试探性删除
        const double delta = (satisfiedSA - satB) * 100.0 + 1.0;
        // delta = sat损失×100 + 1（组数减少的收益）
        // 若 sat不变：delta=+1>0，一定接受（减少了一组）
        // 若 sat下降：delta=-99<0，温度高时仍有概率接受
        if (delta < 0 && uProb(rng) > std::exp(delta / T)) {
            saAdd(groupKSubs[gi]);  // 拒绝→回滚
        } else {
            finalGroups.erase(...);  // 接受→永久删除
            groupKSubs.erase(...);
        }

    } else {  // === Swap ===
        int gi = gD(rng);
        auto ng = randKGroup();
        auto ns = getKSubIndices(ng);
        const int satB = satisfiedSA;
        saRemove(groupKSubs[gi]);   // 先删旧的
        saAdd(ns);                  // 再加新的
        const double delta = (satisfiedSA - satB) * 100.0;
        // sat净变化×100（组数不变，不考虑 size 代价）
        if (delta < 0 && uProb(rng) > std::exp(delta / T)) {
            saRemove(ns);
            saAdd(groupKSubs[gi]);  // 回滚（两步撤销）
        } else {
            finalGroups[gi] = ng;   // 接受
            groupKSubs[gi] = ns;
        }
    }

    // 记录历史最优（以防 SA 后期变差）
    if (satisfiedSA > bestSatisfied || ...) {
        bestSAGroups = finalGroups;  // 拍快照
    }
}
finalGroups = bestSAGroups;  // 恢复历史最优
```

## 第八步：后向消除（`canRemove` lambda）

旧代码与新代码逻辑相似，但旧代码每次都调用 `getKSubIndices`（即重新枚举），新代码预缓存了 `fgKSubs`：

```cpp
// 预缓存每组的 s-子集编号（避免后向消除中重复枚举）
std::vector<std::vector<int>> fgKSubs;
std::vector<int> covRef_be(numSubs, 0);
for (const auto& kg : finalGroups) {
    fgKSubs.push_back(getKSubIndices(kg));
    for (int si : fgKSubs.back()) ++covRef_be[si];
}

auto canRemove = [&](int g) -> bool {
    for (int si : fgKSubs[g]) --covRef_be[si];   // ① 临时减少引用计数
    bool ok = true;
    for (int i = 0; i < numJ && ok; ++i) {        // ② 验证所有 j-子集
        int cnt = 0;
        for (int si : jToSubs[i])
            if (covRef_be[si] >= 1) ++cnt;         // 仍被至少一组覆盖 → 计数
        if (cnt < minCover_) ok = false;            // 不满足 → 不能删
    }
    for (int si : fgKSubs[g]) ++covRef_be[si];    // ③ 无论结果，还原引用计数
    return ok;
};

// 反复扫描，找到可删的就删
bool improved = true;
while (improved) {
    improved = false;
    for (int g = 0; g < (int)finalGroups.size(); ++g) {
        if (canRemove(g)) {
            for (int si : fgKSubs[g]) --covRef_be[si];  // 永久减少
            finalGroups.erase(...);
            fgKSubs.erase(...);
            improved = true;
            break;   // 重新从头开始（删除后其他组可能变得可删）
        }
    }
}
```

具体执行（示例1，6个候选 k-组）：

```
尝试删除第0组（{1,2,3,4,5,6}）：
  临时减少其6个 s-子集的 covRef_be
  检查所有21个 j-子集：
    j-子集{1,2,3,4,5}（编号0）：jToSubs[0]={0}，covRef_be[0]现在=0（只有第0组覆盖它）
    cnt=0 < minCover_=1 → ok=false
  还原 → 不能删第0组

尝试删除第1组（{1,2,3,4,5,7}）：同理，{1,2,3,4,7} 只有第1组覆盖 → 不能删
...

结论：6组都不可删，结果保持6组 ✓
```
