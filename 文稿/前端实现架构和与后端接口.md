## 1. 整体架构概览

本项目是一个基于 **Tauri 2** 的跨平台桌面应用。整体分为三层：

```
┌─────────────────────────────────────────────────────────┐
│                  前端（React + TypeScript）               │
│   App → [Generate | DBBrowser | PoolManager]            │
│   src/lib/tauri.ts  ← 所有 invoke 调用的统一出口         │
└────────────────────┬────────────────────────────────────┘
                     │  Tauri invoke（IPC）
┌────────────────────▼────────────────────────────────────┐
│              Rust 中间层（Tauri Commands）                │
│   src-tauri/src/commands.rs                             │
│   - 文件系统操作（list/read/delete）                     │
│   - Shell Sidecar 调用（生成任务）                       │
│   - 对话框 Plugin（import_pool）                         │
└────────────────────┬────────────────────────────────────┘
                     │  进程通信（stdout JSON）
┌────────────────────▼────────────────────────────────────┐
│              C++ Sidecar（optimal_sample）               │
│   code/main.cpp + SampleSelectSystem + ILPSolver        │
│   --cli 模式：读参数 → 执行算法 → 输出 JSON              │
└─────────────────────────────────────────────────────────┘
                     │  文件读写
┌────────────────────▼────────────────────────────────────┐
│              数据目录（dbDir）                            │
│   {AppData}/db/                                         │
│   ├── *.txt          → 结果 DB 文件                     │
│   └── pool_*.txt     → 号码池文件                       │
└─────────────────────────────────────────────────────────┘
```

## 4. 前端页面与组件

### 4.1 App.tsx（根组件 / 布局）

**职责：** 应用外壳，包含左侧导航栏和右侧内容区。

**布局结构：**
```
┌──────────────┬────────────────────────────────────────────┐
│  左侧导航栏   │            右侧内容区（页面渲染）             │
│  (w-56)      │                                            │
│  ─────────   │   根据 page 状态条件渲染：                   │
│  Configure   │   - page === "generate" → <Generate />     │
│  Results     │   - page === "db"       → <DBBrowser />    │
│  Pool Mgr    │   - page === "pool"     → <PoolManager />  │
│  ─────────   │                                            │
│  v0.1.0      │                                            │
└──────────────┴────────────────────────────────────────────┘
```

**关键状态：**

| 状态 | 类型 | 初始值 | 说明 |
|------|------|--------|------|
| `page` | `"generate" \| "db" \| "pool"` | `"generate"` | 当前激活页 |
| `dbDir` | `string` | `""` | 数据目录路径，通过 Tauri Path API 解析 |

**数据目录初始化（非 invoke）：**

```typescript
// 使用 @tauri-apps/api/path 的原生 API，不走 invoke
useEffect(() => {
  appDataDir()
    .then((dir) => join(dir, "db"))
    .then(setDbDir)
    .catch(() => setDbDir("./db"));  // 失败时回退为相对路径
}, []);
```

`dbDir` 作为 prop 向下传入三个子页面。

### 4.2 Generate.tsx（配置与生成页）

**职责：** 配置算法参数，选择样本模式，调用 C++ 后端生成最优 k-组，展示结果。

**布局：** 左侧参数面板（w-72）+ 右侧结果展示区（flex-1）

**参数配置项：**

| 参数 | 标签 | 范围 | 说明 |
|------|------|------|------|
| `m` | m (total samples) | 45 ~ 54 | 总样本数（彩票号码池大小） |
| `n` | n (selected samples) | 7 ~ 25 | 选取的样本数量 |
| `k` | k (group size) | 4 ~ 7 | 每个 k-组的大小，默认 6 |
| `j` | j (cover subset size) | s ≤ j ≤ k | 覆盖子集大小 |
| `s` | s (min intersection) | 3 ≤ s ≤ j | 最小交集大小 |
| `minCover` | minCover (min coverage) | 1 ~ 20 | 最小覆盖次数，通常为 1 |

**样本选取模式（SampleMode）：**

| 模式 | 说明 | 对应参数 |
|------|------|---------|
| `random` | 随机选取 n 个样本 | `samples` 字段不传 |
| `manual` | 手动输入 n 个数字（空格或逗号分隔） | `samples` 传入解析后的数组 |
| `pool` | 从已导入的 pool 文件中选取 | `poolFile` 传入文件路径 |

**本地状态：**

| 状态 | 类型 | 说明 |
|------|------|------|
| `params` | `Params` | 当前参数值 |
| `sampleMode` | `SampleMode` | 样本选取模式 |
| `manualInput` | `string` | 手动输入的数字字符串 |
| `selectedPool` | `string` | 选中的 pool 文件路径 |
| `pools` | `PoolFileInfo[]` | 可用 pool 文件列表 |
| `runCount` | `number` | 保存时的 run 编号 |
| `autoSave` | `boolean` | 是否自动保存到 DB |
| `loading` | `boolean` | 生成中状态 |
| `result` | `GenerateResult \| null` | 生成结果 |
| `error` | `string` | 错误信息 |

**invoke 调用：**
- 页面加载时：`listPools(dbDir)` → 填充 pool 下拉列表
- 点击 Generate：`runGenerate(params)` → 获取生成结果

**结果展示：**
- Selected Samples 卡片：展示 n 个选中样本（Badge 形式）
- Optimal k-Groups 卡片：`<GroupTable>` 展示所有 k-组，样本集中的数字高亮显示

### 4.3 DBBrowser.tsx（结果数据库浏览页）

**职责：** 浏览、查看、删除已保存的结果 DB 文件（`.txt`）及其中的 k-组。

**布局：** 左侧文件列表（w-64）+ 右侧 k-组内容区（flex-1）

**本地状态：**

| 状态 | 类型 | 说明 |
|------|------|------|
| `files` | `DbFileInfo[]` | DB 文件列表 |
| `loadingList` | `boolean` | 文件列表加载中 |
| `selectedFile` | `DbFileInfo \| null` | 当前选中文件 |
| `groups` | `number[][]` | 当前文件的 k-组数据 |
| `loadingGroups` | `boolean` | k-组数据加载中 |
| `deletingFile` | `boolean` | 文件删除中 |

**invoke 调用：**
- 页面加载 / 点击刷新：`listDb(dbDir)` → 刷新文件列表
- 点击文件：`readDb(file.path)` → 读取 k-组内容
- 点击 Delete File：`deleteDb(file.path)` → 删除整个文件
- 点击行删除按钮：`deleteGroup(file.path, index)` + `readDb()` → 删除单个 k-组并刷新

## 7. 数据目录约定（dbDir）

应用启动时，`App.tsx` 通过 Tauri Path API（**非 invoke**）确定数据目录：

```typescript
import { appDataDir, join } from "@tauri-apps/api/path";

appDataDir()                    // macOS: ~/Library/Application Support/com.optimal-sample.app/
  .then((dir) => join(dir, "db"))
  .then(setDbDir)
  .catch(() => setDbDir("./db"));  // 回退：相对路径
```

**目录内文件约定：**

| 文件名模式 | 类型 | 说明 |
|-----------|------|------|
| `*.txt`（非 pool_ 开头） | DB 文件 | 保存算法结果，每行一个 k-组，逗号分隔整数 |
| `pool_*.txt` | Pool 文件 | 号码池，逗号分隔整数，`#` 开头为注释行 |

**DB 文件命名示例：**`45-8-6-6-5-1-4.txt`（参数：m-n-k-j-s-minCover-run）

## 8. 前后端接口层（src/lib/tauri.ts）

所有前端与 Rust 后端的通信均通过 `src/lib/tauri.ts` 封装，使用 `@tauri-apps/api/core` 的 `invoke`。

### 8.1 类型定义

#### GenerateParams（生成参数）

```typescript
export interface GenerateParams {
  m: number;           // 总样本数（45~54）
  n: number;           // 选取样本数（7~25）
  k: number;           // k-组大小（4~7）
  j: number;           // 覆盖子集大小（s≤j≤k）
  s: number;           // 最小交集（3≤s≤j）
  minCover: number;    // 最小覆盖次数
  samples?: number[];  // 手动指定样本（可选；不传则随机选取）
  save: boolean;       // 是否保存到 DB 文件
  run: number;         // 保存时的 run 编号（用于文件名）
  dbdir: string;       // 数据目录绝对路径
  poolFile?: string;   // 使用 pool 模式时的文件路径（可选）
}
```

#### GenerateResult（生成结果）

```typescript
export interface GenerateResult {
  success: boolean;    // 是否成功
  count: number;       // 生成的 k-组数量
  dbFile: string;      // 保存的 DB 文件路径（未保存时为空字符串）
  samples: number[];   // 实际使用的 n 个样本
  groups: number[][];  // 所有生成的 k-组（二维数组）
  error?: string;      // 错误信息（仅 success=false 时存在）
}
```

#### DbFileInfo（DB 文件信息）

```typescript
export interface DbFileInfo {
  path: string;    // 文件绝对路径
  name: string;    // 文件名（含 .txt 后缀）
  label: string;   // 显示标签（去除 .txt 后缀的文件名）
}
```

#### PoolFileInfo（Pool 文件信息）

```typescript
export interface PoolFileInfo {
  path: string;       // 文件绝对路径
  name: string;       // 文件名（pool_*.txt）
  numbers: number[];  // 文件中的所有整数
}
```

### 8.3 各接口详细说明

#### runGenerate

触发 C++ 核心算法，是最关键的接口。

```typescript
// 调用示例（Generate.tsx）
const res = await runGenerate({
  m: 45, n: 8, k: 6, j: 6, s: 5, minCover: 1,
  samples: undefined,          // random 模式不传
  save: true,                  // 保存到文件
  run: 1,                      // 第 1 次运行
  dbdir: "/path/to/db",
  poolFile: undefined,         // 不使用 pool
});
// res.success === true 时 res.groups 包含所有 k-组
```

**三种调用模式对比：**

| 模式 | samples | poolFile | 说明 |
|------|---------|----------|------|
| random | 不传（`undefined`） | 不传 | C++ 内部随机选取 n 个样本 |
| manual | `[1, 5, 12, 23, ...]` | 不传 | 使用前端指定的 n 个样本 |
| pool | 不传 | `"/path/pool_xxx.txt"` | C++ 从 pool 文件中选取样本 |

#### listDb / readDb

```typescript
// 列出文件
const files = await listDb("/path/to/db");
// files: [{ path: "/path/to/db/45-8-6-6-5-1-1.txt", name: "45-8-6-6-5-1-1.txt", label: "45-8-6-6-5-1-1" }]

// 读取内容
const groups = await readDb("/path/to/db/45-8-6-6-5-1-1.txt");
// groups: [[1,5,12,23,31,45], [2,8,15,27,33,44], ...]
```

#### deleteGroup

`index` 为 0-based，Rust 侧直接按行号删除并重写文件。

```typescript
await deleteGroup("/path/to/db/file.txt", 2);  // 删除第 3 行（索引 2）
```

#### importPool

调用系统原生文件选择对话框（由 Rust 侧 `tauri_plugin_dialog` 处理），用户选择 `.txt` 文件后：
- 若文件名不以 `pool_` 开头，Rust 自动添加前缀
- 复制到 `dbDir` 目录下
- 返回 `PoolFileInfo`；用户取消选择返回 `null`

```typescript
const pool = await importPool("/path/to/db");
if (pool) {
  // pool.name === "pool_myfile.txt"（自动添加前缀）
  // pool.numbers === [1, 5, 12, 23, ...]
}
```

## 9. Tauri 后端（Rust）命令实现

### 9.1 命令注册

`src-tauri/src/lib.rs` 注册所有插件和命令：

```rust
pub fn run() {
    tauri::Builder::default()
        .plugin(tauri_plugin_shell::init())    // Sidecar 进程支持
        .plugin(tauri_plugin_dialog::init())   // 文件选择对话框
        .plugin(tauri_plugin_fs::init())       // 文件系统权限
        .invoke_handler(tauri::generate_handler![
            run_generate,    // 调用 C++ 生成
            list_db,         // 列出 DB 文件
            read_db,         // 读取 DB 文件
            delete_db,       // 删除 DB 文件
            delete_group,    // 删除单个 k-组
            list_pools,      // 列出 pool 文件
            read_pool,       // 读取 pool 文件
            import_pool,     // 导入 pool 文件
        ])
        .run(tauri::generate_context!())
        .expect("error while running tauri application");
}
```

---

### 9.2 run_generate（调用 C++ Sidecar）

**核心流程：**

```
1. 确保 dbdir 目录存在（create_dir_all）
2. 构建命令行参数列表：
   --cli
   --m={m}  --n={n}  --k={k}  --j={j}  --s={s}
   --minCover={minCover}
   --run={run}
   --dbdir={dbdir}
   [--samples={n1,n2,...}]    // 仅 manual 模式
   [--save]                   // 仅 autoSave=true
   [--poolFile={path}]        // 仅 pool 模式

3. 调用 Shell Sidecar（optimal_sample 二进制文件）
4. 等待进程退出，取 stdout
5. 从 stdout 中提取最后一个以 '{' 开头的行（JSON）
6. 反序列化为 GenerateResult 并返回
```

**Sidecar 调用关键代码（Rust）：**

```rust
let output = app
    .shell()
    .sidecar("optimal_sample")   // 对应 tauri.conf.json 中 bundle.externalBin
    .args(&args)
    .output()
    .await?;

let stdout = String::from_utf8_lossy(&output.stdout);
let json_str = extract_json(&stdout);  // 取最后一个 '{' 开头的行
let result: GenerateResult = serde_json::from_str(json_str)?;
```

**Rust 数据结构（与前端 TypeScript 类型一一对应）：**

| Rust 字段 | TS 字段 | serde rename | 类型 |
|-----------|---------|-------------|------|
| `min_cover` | `minCover` | `#[serde(rename = "minCover")]` | i32 |
| `pool_file` | `poolFile` | `#[serde(rename = "poolFile")]` | `Option<String>` |
| `db_file` | `dbFile` | `#[serde(rename = "dbFile")]` | String |

---

### 9.3 list_db / read_db / delete_db / delete_group

所有操作均为纯文件系统操作（`std::fs`），不涉及 C++：

**list_db：** 遍历 `dbDir`，筛选 `is_db_file`（以 `.txt` 结尾且不以 `pool_` 开头），按名称排序返回。

**read_db：** 逐行读取文件，跳过空行和 `#` 注释行，每行按逗号分割解析为 `Vec<i32>`。

**delete_db：** 直接 `fs::remove_file`。

**delete_group：** 读取全部 k-组 → 按索引删除 → 重写文件（每组一行，逗号连接）。

---

### 9.4 list_pools / read_pool / import_pool

**list_pools：** 遍历 `dbDir`，筛选 `pool_*.txt` 文件，读取每个文件的数字内容，按名称排序返回。

**read_pool：** 逐行读取，逗号分割，收集所有整数（`#` 开头跳过）。

**import_pool 流程：**

```
1. 弹出系统文件选择器（tauri_plugin_dialog），过滤 .txt 文件
2. 用户取消 → 返回 None
3. 获取源文件路径
4. 若文件名不以 pool_ 开头 → 目标名 = "pool_" + 原文件名
5. create_dir_all(db_dir)
6. fs::copy(src_path, db_dir/dest_name)
7. 读取目标文件数字 → 返回 PoolFileInfo
```

## 10. C++ Sidecar 接口协议

### 命令行参数

| 参数 | 类型 | 必须 | 说明 |
|------|------|------|------|
| `--cli` | flag | ✅ | 启用 CLI 模式（不显示交互菜单） |
| `--m=N` | int | ✅ | 总样本数 |
| `--n=N` | int | ✅ | 选取样本数 |
| `--k=N` | int | ✅ | k-组大小 |
| `--j=N` | int | ✅ | 覆盖子集大小 |
| `--s=N` | int | ✅ | 最小交集大小 |
| `--minCover=N` | int | ✅ | 最小覆盖次数 |
| `--run=N` | int | ✅ | 运行编号（用于文件名） |
| `--dbdir=PATH` | string | ✅ | 数据目录路径 |
| `--samples=N1,N2,...` | int[] | ❌ | 手动指定样本（逗号分隔） |
| `--save` | flag | ❌ | 保存结果到 DB 文件 |
| `--poolFile=PATH` | string | ❌ | 从 pool 文件中选取样本 |

### stdout 输出格式

C++ 在 `--cli` 模式下，将最终结果作为 **单行 JSON** 输出到 stdout。Rust 侧提取最后一个以 `{` 开头的行进行解析。

**成功时：**

```json
{
  "success": true,
  "count": 12,
  "dbFile": "/path/to/db/45-8-6-6-5-1-1.txt",
  "samples": [3, 7, 12, 18, 24, 31, 39, 45],
  "groups": [
    [3, 7, 12, 18, 24, 31],
    [3, 7, 12, 18, 24, 39],
    ...
  ]
}
```

**失败时：**

```json
{
  "success": false,
  "count": 0,
  "dbFile": "",
  "samples": [],
  "groups": [],
  "error": "错误描述信息"
}
```

> **注意：** C++ 的交互式菜单、ILP 求解器等功能不暴露给前端，GUI 只使用 `--cli` 模式这一条通信路径。

## 12. 完整数据流图

### 生成流程（Generate）

```
用户配置参数 & 点击 Generate
        │
        ▼
Generate.tsx: handleGenerate()
  - 参数验证（validate()）
  - 构建 GenerateParams 对象
        │
        ▼ invoke("run_generate", { params })
src/lib/tauri.ts: runGenerate(params)
        │
        ▼ Tauri IPC
commands.rs: run_generate()
  - create_dir_all(dbdir)
  - 构建 CLI 参数数组
        │
        ▼ Shell Sidecar
binaries/optimal_sample --cli --m=45 --n=8 ...
  - C++ 执行贪心/ILP 算法
  - stdout: {"success":true,"count":12,"samples":[...],"groups":[[...],...]}
        │
        ▼ parse stdout JSON
commands.rs → GenerateResult
        │
        ▼ Tauri IPC return
Generate.tsx: setResult(res)
  - 渲染 Selected Samples Card
  - 渲染 GroupTable（高亮样本集数字）
```

### DB 管理流程（DBBrowser）

```
页面加载
  │ invoke("list_db", { dbDir })
  ▼
files: DbFileInfo[]（文件列表）
  │
  │ 用户点击文件 → invoke("read_db", { path })
  ▼
groups: number[][]（k-组列表） → GroupTable
  │
  ├── 用户点击 Delete File → invoke("delete_db", { path }) → 刷新列表
  └── 用户点击行删除 → invoke("delete_group", { path, index }) → 重新 readDb
```

### Pool 管理流程（PoolManager）

```
页面加载
  │ invoke("list_pools", { dbDir })
  ▼
pools: PoolFileInfo[]

用户点击 Import Pool File
  │ invoke("import_pool", { dbDir })
  ▼
Rust: 系统文件选择对话框
  │ 用户选择 .txt 文件
  ▼
Rust: 复制到 dbDir（自动加 pool_ 前缀）
  │ 返回 PoolFileInfo
  ▼
前端: 刷新列表 & 自动选中新导入的 pool
```
