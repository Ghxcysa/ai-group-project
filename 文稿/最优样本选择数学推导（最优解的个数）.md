# 最优样本选择的流程
系统输入参数：
- m：总样本数
- n：从 m 选 n
- k：输出 group 大小
- j：检查的组合大小
- s：必须覆盖的样本数

流程是：

m → 选 n → 构造 k-groups → 满足 j,s 条件

# 数学模型的基本定义

集合：N={1,2,3,...,n}

所有可能的 k-groups : $K_i\subseteq N,|K_i|=k$

 决策变量

$x_i = \begin{cases} 1 & 选择该 k-group \\ 0 & 不选择 \end{cases}$

目标：最少的k group
$min\sum x_i$  
约束：
对每个 s-subset：
$S \subseteq N, |S|=s$

必须存在一个 $K_i$​

使得：
$S \subseteq K_i$
​
即每个 s-subset 至少被一个 k-group 覆盖。

# 理论下界推导

#### 1. 计算所有的 s-subsets
从n个元素中选s:  $C_n^s$ 

#### 2.每个k-subset 覆盖的s
一个k set 可以选 $C_k^s$ 个subset

#### 3.理想情况
假设：
- 每个 k-group 覆盖 **完全不同** 的 s-subsets
- 没有任何重复

最少group ： $$t \ge \frac{C_k^s}{C_n^s}$$

但是不准确，有没有更准确的计算方法？

# Schönheim Bound 推导

核心思路-- 统计一个元素被包含了多少次

在 n 集合里，如果一个 s-subset 包含 x， 那么就剩下了 s-1个元素，从n-1 里面选
所以 ： $C_{n-1}^{s-1}$ 
在k集合： $C_{k-1}^{s-1}$ 
最少group:
$$t \ge \frac{C_{k-1}^{s-1}}{C_{n-1}^{s-1}}$$

如果选择了 t个 k-set
那么总元素出现的次数： t x k
平均每个元素出现次数 ： tk/n
所以 
$$t \ge [\frac{n}{k}\frac{C_{k-1}^{s-1}}{C_{n-1}^{s-1}}]$$
递归展开得到 ： $C(n-1,k-1,s-1) => C(n-s+1,k-s+1,1)=[\frac{m}{r}]$ 

Schönheim Bound
$C(n,k,s)≥⌈kn​C(n−1,k−1,s−1)⌉$
$C(n,k,s)≥⌈kn​⌈k−1n−1​⌈k−2n−2​⋯⌈k−s+1n−s+1​⌉⌉⌉⌉$
